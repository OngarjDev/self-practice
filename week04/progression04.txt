เป็นไฟล์ที่ผมลองฝึกโจทย์ยากใน LeetCode เกี่ยวกับ sliding Windows
โจทย์นี้ผมแก้ไม่ได้ครับ แต่ผมได้เรียนรู้วิธีการทำ sliding windows 
ที่ผมเข้าใจคือ 

วิธีที่ผมทำคือ 
1. ผมแยก parameter t  เอาคำที่ต้องการหาออกมา เช่น ABCA  เราต้องหาประโยค ผมใช้ spend แยกคำ
2. แนวคิดคือให้มันไล่ที่ละตัวเหมือนกัน แต่ถ้ามันเจอ ให้มันลบตัวอักษรที่เจอใน obj_have
3. ถ้า obj_have ลบหมดแล้ว
4. ตรวจสอบว่ามันสั้นกว่าไหม ถ้าสั้นกว่าผมบันทึกตำแหน่ง แลพ ข้อความเอาไว้ครับ
5. รีเซ็ตค่า obj_have(ขั้นตอนที่1) ให้ลูปหาใหม่ครับ

ส่วนเฉลย จาก chat gpt
ในช่วงแรกผมงงมากผมดู Debug หลายวันเลย
1. มันจะแยกคำที่ต้องการหาก่อน เก็บในตัวแปร เช่น {A:1,B:2} มันต้องหาให้ครบ
2. มันจะมี pointer ขวา ที่เลื่อนทีละตำแหน่ง โดยสรุปเลยมันจะเลื่อนไปเรื่อยจนถึงตัวสุดท้าย
3. ระหว่างการเลื่อนมันก็จะเช็คค่าไปทุกครั้งว่ามันมีครบไหม ถ้าครบ1ตัว satisfiedCount จะ + 1
4. ถ้า satisfiedCount ครบตามที่ต้องการแล้ว
    4.1 มันจะตรวจสอบความยาวว่ามันน้อยกว่าค่าเดิมไหม ถ้าสั้นกว่ามันจะตัด และ บันทึกตำแหน่ง
    4.2 จากนั้นมันขยับ sliding windows Start 1ตำแหน่ง
    สมมุติ ADSDAWDQEQE ต้องการหา ASAD Pointerขวา จะไปทางขวาจนครบทุกตัว
    ส่วนจุดเริ่มต้นเริ่ม(4.2)จาก A พอ Pointer ขวาเจอมันจะขยับไป D -> S -> D จนถึง n
    4.3 พอมันขยับ Pointer ซ้ายแล้ว มันจะลบ ตัวอักษรก่อนหน้าออก เพื่อหาต่อไป จนเจอตัวที่ต้องการครบ
